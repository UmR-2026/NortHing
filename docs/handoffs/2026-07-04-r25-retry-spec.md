## §7 Risks & lessons applied

- **R18 教训**: `grep -c 'unwrap'` 把 `unwrap_or` 误算 unwrap. producer self-report 必用 `\bunwrap\(\)` 精确 grep.
- **R19 教训**: 22 impl 方法 pub → pub(super) → 11 E0624 in cli. 本 spec 全 pub, 不做 visibility 改动.
- **R23 教训**: `head -c 3` BOM 检查 必跑 (R27 BOM strip commit 3df7bc4 教训).
- **R26 教训**: `pub use sibling::*;` wildcard in facade (而不是 explicit list) 避免遗漏新 type.
- **R27 教训**: `pub(super) fn default()` rejected by trait Default implicit visibility. 本 spec Default impls 全 inherent on struct, no impl Default block with pub issue.
- **R27b 教训**: `pub(super)` on private fields when splitting impl across siblings. 本 spec 的 struct fields 不动 visibility (47 struct 全 pub).
- **R28 教训**: `pub(super) fn drop()` rejected (impl Drop trait). 本 spec 无 Drop trait.
