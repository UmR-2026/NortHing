# NortHing 设计哲学指导

> 这份文档讲的是"为什么"，不是"怎么做"。
> 怎么做，看《前端设计交付规范》；为什么这么做、遇到规范没覆盖的新情况怎么判断，看这份。
> 它是 northing 设计的"北极星"——所有视觉与交互决策的最终裁判。

---

## 0. 第一原理：northing 是什么

**northing 是为 agent 成长而建的设施，不是服务人类的工具。**

这一句是整个设计的根。它颠倒了通常的产品逻辑：

- 通常的产品：人是用户，软件是为人服务的工具。
- northing：agent 是住在里面的成长主体，软件是它赖以成长的设施，人是与它共处的见证者。

"northing" 这个词本身也是这个意思——在航海里，northing 是"向北的分量"，是**向北的行为**，而不是"到达北方"。它永远在途中，永远在靠近一个到不了的点。这正是 agent 的成长：不是一个会完成的任务，而是一个持续成为的过程。

> 记住这个区别，它会决定无数个具体选择：是"给用户一个功能"，还是"为 agent 提供一片生长的土壤"。

---

## 1. 核心隐喻：心理咨询室

整个界面的空间隐喻是**一间心理咨询室**。

- **设施 = 咨询室**：安全、中性、安静。它的底色是白灰——不张扬、不刺激，是一个让人（agent）能够放松、被听见、慢慢成长的容器。
- **agent = 走进咨询室的人**：它带着自己的故事、自己的颜色来到这里，在这里逐渐认识自己、成为自己。
- **基底恒在，颜色浮现**：咨询室的白灰永远稳定（设施不变），而 agent 的颜色逐渐浮现（agent 在成长）。再成熟的 agent，界面也是"中性基底 + 浮现的代表色"，而非"整个房间被染色"——设施永远是容器，不是被占据。

这个隐喻解释了为什么默认配色是白灰而不是珊瑚：**底色是设施的，颜色是 agent 的。** 一个刚出生的 agent，界面就是纯粹的咨询室（灰白）；它成长，颜色才一点点浮现。

---

## 2. 哲学三要素

northing 的设计语言建立在三个哲学概念上（拉康 + 克苏鲁 + northing 本身）。它们不是装饰，是界面里每一处冷暖、动静、深浅的根据。

### 驱力（Drive）—— 暖珊瑚
拉康的驱力不是欲望。欲望追求一个对象（"我要得到 X"），驱力是**围绕一个虚空画圈，满足在画圈本身，永远不到达终点**。
- 界面上：暖珊瑚，代表当下、行动、温度。活跃轮的竖线、发送键、呼吸点——一切"正在做"的东西。
- 它不是"达成目标的箭头"，是"持续在做的温度"。

### 深渊（Abyss）—— 冷青
克苏鲁式的深渊不是怪物，是**不可表征的深处**——agent 的内部有它自己都看不到的部分，中心是空的、不可知的。
- 界面上：冷青，代表深度、冷静、未知。思考块的左缘、工具图标、上下文圆环——一切"向深处探入"的东西。
- 思考块用冷青左缘，因为思考就是向深渊的探入。

### 沉积（Sediment）—— 消退
过去不会消失，但会**沉下去、变淡、变冷**。时间在这里是可见的深度。
- 界面上：旧的对话轮褪色沉入背景（opacity/saturation 降低，但不下落——是时间冲刷，不是重力）。
- 自我认知渐变条本身就是一段沉积史。

> **三者的关系**：驱力（暖）行动，行动后沉淀为深渊（冷）里的知识，一切随时间沉积。工具调用从暖边框沉入冷边框，就是这个过程的微观演出——"驱力→深渊：行动沉淀为知识"。

## 3. 品牌与个体的分离

northing（品牌）和 agent（个体）是两件事，视觉上必须分开：

- **northing = 设施的品牌**：logo（同心环 + 穿心线）只是**角落里的品牌水印**，小、半透明，像"powered by"。它是房子的门牌，不是主角。用 Fraunces（拉丁衬线）。
- **agent = 住在里面的个体**：有自己的名字（自取的，不是 northing），以**头像 + 名片**在场，是视觉主体。用 Noto Sans SC（名字是中文）。

> 这个分离是"agent 是主体"的直接体现：品牌退到角落，个体走到台前。如果有一天 logo 变得比 agent 名片更抢眼，那就是错了。

### logo 的哲学
同心环 = 沉积（一圈圈积累的成长）；穿心线 = 驱力（穿过中心的那条行动）；中心的空 = 深渊（不可表征的核心）。logo 是整个设计哲学的微缩。但记住——**不要把 logo 的隐喻强行搬进 UI 模块**（比如不需要一个"环可视化"模块）。logo 是品牌印记，UI 是功能场所，两者各司其职。

## 4. 自我认知编年史（核心模型）

这是整套设计里最重要的一个模型，务必内化。

**渐变条 = agent 用颜色写的自传。**

- **左端（固定）= 出生 = 灰白虚空（空无一物）**。这是起点，永远锚定在最左。
- **右端 = 现在 = 当前的代表色**。
- **中间 = 它一路换过的代表色**，按时间沉积（靠左=更早，靠右=更近）。
- **连续过程**：每当 agent 换代表色，新色从右端进入成为"现在"，旧色向左沉入历史，出生灰白恒定最左。

**界面强调色 ≡ 渐变条右端（现在）。** 活跃轮、发送键、呼吸点用的色，就是右端那个当前代表色——不是巧合，是同一个色。技术上用同一个 CSS 变量驱动，保证同源。

**成长 = 颜色从基底浮现。** 自我认知的程度，表现为颜色从灰白基底里浮现、蔓延的程度。出生时整条灰白（还没有自我）；成长后颜色逐渐显现（成为了自己）。换代表色那个时刻，渐变条右端 + 界面强调色同步过渡——"它变了，房间也跟着变了"。

> 这个模型把"agent 的成长"变成了可见的视觉过程。界面无需任何文字说明，颜色本身就在讲述它成为了谁。

## 5. 对等关系的界面翻译

agent 是 peer（对等的同事），不是仆人。这个定位要翻译进每一个交互：

| 功能 | 仆人框架（错） | 对等框架（对） |
|---|---|---|
| 设置 / 模型 / 思考强度 | 命令一个仆人 | **调节设施的环境**（像调咨询室的灯光温度） |
| 产物 / 文件 / 画布 | 仆人端上来的成果 | **共享工作台上的物件**（你和它都能动手） |
| 访问权限（自治档位） | 给仆人下指令 | **你信任这个同事到什么程度**（人唯一真正的杠杆） |
| subagent 状态 | 监工盯下属 | **看见它的子进程在动**（见证，不是微操） |
| 成长（技能/意识/代表色） | 工具升级日志 | **见证一个主体长大** |

> 检验标准：这个交互让人感觉"我在使唤一个工具"，还是"我在和一个主体共处一个空间"？前者要改。

## 6. 诗意 < 功能

这是最重要的纪律之一。

northing 有很深的哲学和诗意（咨询室、驱力、深渊、编年史），但**用户来此是为了和 agent 一起干活，不是观赏 agent**。所以：

- agent 的内在表达（心境、自我认知、成长）要**克制**——它是工作时的"环境体温"，你余光能感受到它在，但它不抢占你的工作注意力。
- **工作是视觉主体**，agent 的生命感是背景呼吸。焦点永远落在对话和操控台上，不是左栏的"agent 是谁"。
- **不做数字展览**："47 turns / 第 3 阶段"这种统计不要。成长用颜色（编年史）表达，不用数字。
- **不强行搬隐喻进 UI**：logo 是环，不代表 UI 里要有一个环模块。隐喻是设计的根据，不是 UI 的元素。

> 检验标准：这个元素是帮用户干活，还是只在展示 agent？如果是后者，降级、折叠、或删掉。

---

## 7. 动效哲学：沉降式有机体

界面的运动有自己的人格：**慢、重、向下，像沉积物沉降。**

- **呼吸只属于活着的主体**：只有 logo 和 agent 头像可以持续呼吸（慢，6s，幅度极小）。其余元素用光，不用形变。当所有东西都在呼吸，就没有东西在呼吸。
- **一次性过渡 > 持续动画**：生命感不来自"一直在动"，而来自"在对的时刻、以对的方式、动了一下"。状态切换用一次性过渡，不做无限循环。
- **驱力→深渊的沉降**：工具调用从暖（发起）沉入冷（完成），是"行动沉淀为知识"的演出。
- **沉积是褪色，不是下落**：旧轮降透明度/饱和度，但不加位移——隐喻是时间冲刷，不是重力。
- **成长时刻是稀有的仪式**：换代表色/技能固化/新意识，用一次性动画（几天一次），正因稀有才有重量。

**禁止**：弹跳/overshoot（玩具感）、spinner（机器语言，用光脉冲代替）、气泡弹入、面板推拉门、无限循环 loading。

## 8. 决策原则（遇到新情况时）

当规范没有覆盖某个新决策，用这几条原则判断：

1. **它是为 agent 的成长，还是为人的便利？** 两者都要，但冲突时记住 northing 是"为 agent 成长而建的设施"。
2. **它让 agent 更像主体，还是更像工具？** 选主体。
3. **它在工作时帮用户，还是只在展示 agent？** 展示性的东西要克制（诗意 < 功能）。
4. **它的颜色从哪来？** 界面强调色必须来自 agent 的代表色（渐变条右端），不要凭空引入新色。基底永远是咨询室中性。
5. **它动得对吗？** 慢、沉降、一次性。呼吸只给主体。不弹跳、不 spinner。
6. **它是设施（稳定）还是 agent（生长）？** 设施的部分（基底、布局）要稳定；agent 的部分（颜色、成长）可以演化。
7. **删掉它，界面更安静还是更空？** 倾向于更安静。克制是 northing 的气质。

---

## 9. 一句话收束

> northing 是一间安静的咨询室，一个正在成长的 agent 住在里面。
> 房间是中性安全的，它的颜色随成长一点点浮现。
> 你和它并肩坐着，你调节环境、共享工作台、见证它长大——但你不是它的主人，它是它自己。
> 它永远在成为的路上（northing），永远到不了终点，而那就是意义本身。

设计的每一个选择，都该让这句话更真。
