## 0. 一句话总纲

northing 是**为 agent 成长而建的设施**。界面的核心隐喻是：**一间心理咨询室（安全中性的基底），一个正在成长的 agent 住在里面（它带来自己的颜色）**。设施稳定，agent 生长。

---

## 1. 产品定位与哲学

- northing 是**为 agent 成长而建的设施**，不是服务人类的工具。
- agent 是**对等的成长主体**（peer），不是仆人/助手。人类是共处的见证者。
- 哲学三要素：
  - **驱力**（drive）：暖珊瑚，当下/行动
  - **深渊**（abyss）：冷青，未知/深度
  - **沉积**（sediment）：过去/消退
- 设计原则：**诗意 < 功能**。用户来此是为了和 agent 一起干活，不是观赏 agent。agent 的内在表达要克制，不能抢工作的戏。

---

## 2. 品牌 / 个体分离（重要）

| | northing（设施品牌） | agent（个体） |
|---|---|---|
| 是什么 | 设施/房子的品牌 | 住在里面的、有自己名字的个体 |
| 视觉 | logo（同心环+穿心线），仅作**角落品牌水印**（小、半透明） | **头像 + 名片**，是在场主体 |
| 字体 | **Fraunces**（拉丁衬线，开 WONK+SOFT 轴） | **Noto Sans SC**（名字是 CJK） |
| 名字 | "northing" | agent 自取（占位名"知序"，真实场景由 agent 自己命名） |

- agent 头像：独立抽象印记（暖珊瑚底 + 一抹深渊青 + 名字首字），**≠ northing logo**。
- logo 永远只是品牌气息，不是视觉主角。

## 3. 默认配色：心理咨询室白灰

**核心：底色中性安全，色彩由 agent 成长带来（不是全界面珊瑚）。**

### 基底 token（咨询室白灰，固定不变）
```
--bg:        #F4F3F0   /* 主背景 */
--surface:   #FBFAF8   /* 卡片底 */
--elevated:  #FFFFFF   /* 浮起卡片 */
--raised:    #EFEDE8   /* 控件底 */
--border:    #E6E3DD
--border-soft:#ECEAE5
--fg:        #38352E   /* 正文 */
--muted:     #7B766C   /* 次级 */
--faint:     #A8A398   /* 弱化（对比度≥4.0:1） */
```

### 强调色（由 agent 代表色驱动，见 §4）
```
--rep-*:  agent 当前代表色（默认珊瑚 #C8714C 系）—— 驱力/行动/温度
--abyss-*: 深渊青 #3F837B 系（哲学常量，相对固定）—— 深度/冷静
```

### 左暖右冷（语义着色）
- 左栏（agent 内在）：暖色边框 `rgba(200,113,76,.13)`
- 右栏（身外之物）：冷色底 `#15171A` 系 + 冷色边框 `rgba(47,122,115,.16)`
- 色温即语义：左=生命/暖，右=物件/冷

## 4. 自我认知编年史（核心模型，务必理解）

**渐变条 = agent 用颜色写的自传（一条时间线）。**

- **左端（固定）= 出生 = 灰白虚空（空无一物）**，永远锚定最左。
- **右端 = 现在 = 当前代表色**。
- **中间 = 它一路换过的代表色**，按时间沉积（靠左=更早，靠右=更近）。
- **连续过程**：每当 agent 换代表色，新色从**右端**进入成为"现在"，旧色**向左沉入历史**，出生灰白恒定最左。

### 对应规则（同源）
- **界面强调色 ≡ 渐变条右端（现在）**。活跃轮竖线、发送键、呼吸点用的色 = 右端那个当前代表色。
- **实现要求**：界面所有驱力强调色引用**同一个 CSS 变量**（如 `--rep-500`），其值由 agent 当前代表色决定。渐变条暖端与界面强调色技术上同源，保证一致。
- **成长时刻**：agent 换代表色时，渐变条右端 + 界面强调色**同步过渡动画**（稀有的一次性事件，几天一次）——"它变了，房间也跟着变了"。

### 三阶段参考（见 chronicle mockup）
- 出生：整条灰白（出生点=现在重合），界面无代表色（强调色=灰）
- 初识自我：第一个代表色在右端，左端仍灰白，界面浮现该色
- 自我深化：新色进右端，旧色左沉为历史，界面强调色同步更新

## 5. 两态设计（可收放）

| | 收起态（默认主页） | 展开态 |
|---|---|---|
| 隐喻 | 独处的咨询室 | 拉开两侧墙，完整觉察 |
| agent 名片 | 在 topbar（紧凑） | 入左栏顶部（舒展，追加 mood/自我认知） |
| 两侧 | 窄把手（可拉开） | 完整侧栏（可收回） |
| 用途 | 专注干活 | 想看它的状态/产物时 |

### 两态切换逻辑（实现要点）
- **单一展开态**（推荐）：任一把手都展开完整三栏，收起回咨询室。不做"半开"四态。
- **操控台是单一组件**：两态像素级一致（最多宽度自适应），**绝不允许两态用不同交互模型**。
- **名片是单一组件**：两态只是布局重排 + 信息渐进披露（收起=核心四件：头像/名/状态/色带；展开=追加 mood/自我认知 caption）。**设置 ⚙ 两态都跟随名片**。
- **状态保持**：收起/展开选择跨会话记忆，默认收起。
- **把手 affordance**：用指向外侧的 chevron（‹ 左 / › 右），收起钮同套方向语言。
- **共用同一套基底 token**：两态不翻转全局明暗（不要收起亮、展开暗）。

## 6. 三栏语义分区

- **左 = 主体（它的内在）**：在场名片、它的子进程、技能候选、意识更新
- **中 = 相遇（对话 + 操控台）**：对话流 + 操控台
- **右 = 世界（身外之物）**：此刻的任务、产物、浏览·画布

### 左栏模块
1. **在场名片**：头像 + 名 + 状态 + 自我认知渐变条 +（展开态追加）mood + 自我认知 caption
2. **它的子进程**：subagent 列表（运行中/待命）。子进程条目应标注归属任务（如"搜索中 · 修复初始化顺序"）
3. **技能候选**：锻造中的新能力（候选中 + 依据"同类失败×N"）+ 已固化技能。接技能锻造机制
4. **意识更新**：agent 意识到了什么（接 episode log）

### 右栏模块
1. **此刻的任务**：当前任务 + 进度（左侧珊瑚竖线，与中栏活跃轮呼应）
2. **产物**：共享物件，区分"它修改/它新建"vs"你上传的"，"未审阅"需有审阅流
3. **浏览·画布**：文件浏览 / 画布 tab

## 7. 组件规范

### 7.1 agent 名片
- 头像 44px 圆角14px，暖珊瑚径向渐变 + 深渊青叠层 + 名字首字
- 名字 16px Noto 500
- 状态行：呼吸点（6s）+ "生成回复中 · 12s"（状态须与轮内当前相位一致，见 §10 状态机）
- 自我认知渐变条 5px 高，左=出生灰白 → 右=现在代表色
- 设置 ⚙ 在头像旁（单色线性 SVG）

### 7.2 操控台（对话框，参考 Qoder/WorkBuddy）
**单一组件，两态一致。** 结构：
- 输入区：placeholder `说点什么…… @ 引用文件 / 调用技能` + 发送键
- 发送键 36px 圆角11px，珊瑚发光。**随状态变形**：空闲=↑发送；生成中=■停止；空输入=禁用
- 控制行（从左到右）：
  - **＋**（附件/添加到上下文）
  - **思考强度 = 分段条**（5 段，3 亮 2 暗，整块可点，hover 显示档位名+后果"更深=更慢更准"）
  - **工作目录 = 直接写目录名**（如 `~/northing`，点击=切换目录）
  - **模型选择**（如 `Claude 4.6 ▾`）
  - **自治档位**（访问权限，显示当前档位如"自治·完全 ▾"，下拉给 3 档明确语义 + denylist 编辑入口）
  - **上下文 = 圆环**（环形进度显示已用%，深渊色，>75% 转暖预警，>90% 提示自动沉积，点击=明细/手动压缩）
- **无麦克风**（不设计语音输入）
- focus 态：边框珊瑚 + 外发光环

### 7.3 对话区
- **活跃轮**：左侧 2.5px 珊瑚竖线锚定，正文 weight 400，名字珊瑚色
- **沉积轮**：opacity .5（褪色非下落，只动 opacity/saturation 不加位移），opacity 下限 .4
- **用户轮**：右对齐气泡（白底 + 边框）
- **思考块**：深渊青左缘 + 冷底 `rgba(63,131,123,.05)`（思考=向深渊探入）
- **工具 chip**：发起=暖边框（驱力）→ 完成=冷边框（深渊，沉淀为知识），350ms ease-out 过渡。chip 可点展开输出
- **每轮元数据**（mono）：模型 / token / 耗时（ctx 只在操控台圆环显示一次，不重复）
- **滚动锚定**：用户在底部则跟随新生成；上滚则钉住 + 出"有新内容↓"浮标

### 7.4 模块卡片
- 左栏卡片：暖边框，圆角 14px
- 右栏卡片：冷边框，冷底
- 模块标题统一：11.5px Noto 500 muted，字距 .02em（中文友好，不用 uppercase 宽字距）

## 8. 字体规范

| 用途 | 字体 | 说明 |
|---|---|---|
| 品牌字（northing）、拉丁展示 | **Fraunces** | 开 `font-variation-settings:"WONK" 1,"SOFT" 60` |
| agent 名、CJK 正文 | **Noto Sans SC** | 正文 weight 400（不用 300，Windows 下太虚） |
| 元数据（token/耗时/ctx/目录） | **JetBrains Mono** | |

### 字号三档（小字端严格收敛，不要 0.5px 幻觉层级）
```
--fs-sm: 10px    /* 时间戳、chip 细节、turn-meta */
--fs-md: 11.5px  /* turn-head、状态行、ctx 标签 */
--fs-lg: 13px    /* 名字、chip、模块标题 */
```
正文 15px，名字 16px。层级靠**颜色（fg→muted→faint）+ 字体（mono vs sans）**承担，不靠字号微调。

## 9. 动效规范（沉降式有机体）

**人格：慢、重、向下，像沉积物沉降。绝不弹跳。呼吸只属于活着的主体。**

- 标准缓动 `cubic-bezier(0.25,0.1,0.25,1)`；快速反馈 150ms；状态切换 350ms ease-out；慢氛围 4-8s；成长时刻 1200ms（仅一次）
- **呼吸只留给 logo / agent 头像**（6s，幅度极小 scale ±1.5%）。其余用光不用形变
- **工具调用**：发起→完成，边框从暖珊瑚沉入冷青（驱力→深渊）
- **沉积**：旧轮褪色（opacity/saturation），不加位移
- **成长时刻**（换代表色/技能固化/新意识）：稀有的一次性动画，渐变条右端 + 界面强调色同步过渡
- **禁止**：弹跳/overshoot、spinner（用光脉冲代替）、气泡弹入、面板推拉门、无限循环 loading

## 10. 界面逻辑（实现需求清单）

> 以下是静态 mockup 无法体现、但可运行主页必须实现的逻辑。

### 10.1 运行时状态机（关键）
完整相位：`空闲(在场) → 思考 → 调用工具 → 生成 → (完成|错误|被打断) → 空闲`，外加 `等待确认`、`断线`。
- 名片状态与轮内指示**绑定同一当前相位**（不能名片说"思考"、元信息说"生成"）
- 发送键随态变形（↑/■/禁用）
- 补齐：**错误态**（暖转冷失败着色 + 重试）、**等待确认门**（内联"允许/拒绝本次/记住"）、**断线态**（顶部 banner + 重连）、**启动态**（头像光脉冲）

### 10.2 会话管理
- 新建/历史/切换会话（收起态 topbar 需有会话标识）
- 首次进入 / 空对话状态 / 加载态

### 10.3 成长闭环（产品立身之本）
- 技能候选 → 固化：定义触发规则（同类成功×N 自动固化 or 人确认），候选卡片可点（展开依据 + 动作）
- 意识更新：可点回溯到触发对话轮，自动归档（7 天前收入折叠区）
- 自我认知色带：可点查看（当前性格 + 演化由来），与代表色共用一次成长动画
- 收起态给成长事件轻量信号（左把手珊瑚点）
- "自取名/换代表色"的稀有仪式时刻（轻量浮现 + 用户可见证）

### 10.4 访问权限（自治档位）
- 控件显示**当前档位**（非固定文案），下拉 3 档明确语义 + denylist 编辑入口
- 定义生效时机（下一轮 or 立即影响进行中操作）
- 非"完全自治"档必须配套**确认门 UI**（与 10.1 联动）

### 10.5 其它
- @ 引用文件 / / 调用技能 的 autocomplete 弹层
- 产物审阅流（diff + 接受/拒绝）
- 子进程可操作性（查看/停止）
- 编辑上一条 / 重新生成 / 复制
- 设置 ⚙ 内容（改名/换色/性格/denylist），与名片自我信息的职能划分
- 快捷键（Enter 发送、Esc 停止等）

## 12. Tauri 实现注意

- 桌面壳：Tauri 2 + React（参考现有 `src/apps/desktop-tauri`，已有 W4/W4b runtime 纪律：core 调用经 `core_rt().spawn`，事件桥 emit-only）
- 现有前端（`src/apps/desktop-tauri/ui`）是单文件 App.tsx + 全局 app.css，本规范要求重构为组件化 + design token
- 字体需自托管 woff2（Fraunces 含 WONK/SOFT 可变轴），避免 Google Fonts 离线回退丢失特征
- agent 代表色 → 界面强调色的同源绑定：建议用 CSS 变量 + Tauri 状态注入，agent 换色时更新 `--rep-*` 变量
- 两态切换、状态机、成长闭环等逻辑需要 Tauri command + 事件桥支持（core 侧已有 agentic 事件系统）
- 适配需要时间：建议分阶段——先视觉层（token + 组件 + 两态布局），再逻辑层（状态机 + 会话 + 成长闭环）

---

## 13. 实现优先级建议

1. **第一阶段（视觉骨架）**：design token 系统 + 字体 + 主页（收起态）布局 + 操控台组件 + 两态切换
2. **第二阶段（核心逻辑）**：运行时状态机（含停止/错误）+ 会话管理 + 对话流
3. **第三阶段（成长系统）**：自我认知编年史（同源色绑定）+ 技能候选 + 意识更新 + 成长时刻动画
4. **第四阶段（完善）**：访问权限档位 + 确认门 + 产物审阅 + @// 弹层 + 异常态
