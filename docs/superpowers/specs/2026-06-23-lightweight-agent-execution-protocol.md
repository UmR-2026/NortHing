# Lightweight Agent Execution Protocol (LAEP)

> **Status:** Approved for implementation
> **Date:** 2026-06-23
> **Author:** ZCode session
> **Version:** 1.0

---

## 1. 核心原则

**三模型分离，各负其责：**

| 模型 | 职责 | 输入 | 输出 |
|------|------|------|------|
| **编码模型** | 按 Taskfile 执行编码 | Taskfile + 上下文 | `change-log.json` |
| **测试模型** | 验证编译/测试/边界 | `change-log.json` + 代码 | `verification-report.json` |
| **Review 模型** | 记录变更，生成 Review Guide | `change-log.json` + `verification-report.json` | `review-guide.md` |

**关键约束：**
- Review 模型不重新验证，只读取前两模型的输出
- 编码模型和测试模型可以是同一轻模型的不同阶段，也可以是不同实例
- 所有输出均为结构化文件（JSON/Markdown），机器可读

---

## 2. Taskfile 格式

每个任务一个 `Taskfile.toml`，定义编码模型的执行边界：

```toml
# Taskfile.toml 模板
[meta]
name = "任务名称"
estimated_time = "2h"
# 可选：指定需要加载的技能
skill = "reference-library"

[context]
# 编码模型必须读取的文件（行号范围可选）
read = [
  "src/apps/desktop/src/app_state/sessions.rs:40-180",
  "src/apps/desktop/src/ui/views/ChatView.slint",
]
# 参考文档（来自 .agents/reference/）
references = [
  "session/06-app-state-slint-wiring.rs",
]

[boundaries]
# 可修改的文件列表
can_modify = [
  "src/apps/desktop/src/app_state/mod.rs",
  "src/apps/desktop/src/app_state/sessions.rs",
]
# 禁止触碰的文件/目录
no_touch = [
  "src/crates/assembly/core/src/agentic/coordination/*.rs",
  "src/crates/execution/agent-dispatch/src/*.rs",
]

[verification]
# 验证命令（测试模型执行）
commands = [
  "cargo check -p northhing --lib",
  "cargo test -p northhing --lib -- --test-threads=1",
]
# 预期结果描述
expected = "0 warnings, all tests pass"

[output]
# 编码模型输出
change_log = ".task/change-log.json"
# 测试模型输出
verification_report = ".task/verification-report.json"
# Review 模型输出
review_guide = "docs/reviews/YYYY-MM-DD-<task>-review.md"
```

## 3. 编码模型输出：change-log.json

编码模型完成编码后，生成结构化变更记录：

```json
{
  "schema_version": "1.0",
  "task": "任务名称",
  "head_before": "abc1234",
  "head_after": "def5678",
  "timestamp": "2026-06-23T10:00:00Z",
  "files_changed": [
    {
      "path": "src/apps/desktop/src/app_state/mod.rs",
      "change_type": "modify",
      "lines_added": 15,
      "lines_removed": 0,
      "description": "添加 current_streaming_session 字段"
    }
  ],
  "tests_added": [
    "build_messages_model_streaming_on_last_assistant"
  ],
  "tests_modified": [],
  "boundary_check": {
    "no_touch_violations": [],
    "can_modify_compliance": true
  },
  "notes": [
    "所有修改在 can_modify 范围内"
  ],
  "confidence": "high",
  "uncertainties": []
}
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `schema_version` | string | 协议版本 |
| `task` | string | 任务名称（来自 Taskfile） |
| `head_before` | string | 编码前 HEAD |
| `head_after` | string | 编码后 HEAD |
| `timestamp` | string | ISO 8601 时间戳 |
| `files_changed` | array | 变更文件列表 |
| `tests_added` | array | 新增测试名列表 |
| `tests_modified` | array | 修改测试名列表 |
| `boundary_check` | object | 边界检查结果 |
| `notes` | array | 编码模型备注 |
| `confidence` | string | `high` / `medium` / `low` |
| `uncertainties` | array | 不确定点（触发 Review 关注） |

## 4. 测试模型输出：verification-report.json

测试模型读取 `change-log.json`，执行验证命令，输出报告：

```json
{
  "schema_version": "1.0",
  "task": "任务名称",
  "timestamp": "2026-06-23T10:05:00Z",
  "status": "PASS",
  "checks": [
    {
      "check": "compile",
      "command": "cargo check -p northhing --lib",
      "status": "PASS",
      "warnings": 0,
      "errors": 0,
      "details": ""
    },
    {
      "check": "test",
      "command": "cargo test -p northhing --lib",
      "status": "PASS",
      "tests_run": 15,
      "tests_passed": 15,
      "tests_failed": 0,
      "details": ""
    },
    {
      "check": "boundary",
      "status": "PASS",
      "no_touch_violations": [],
      "can_modify_compliance": true,
      "details": ""
    }
  ],
  "recommendation": "READY_FOR_REVIEW"
}
```

**状态定义：**

| `status` | 含义 | 下一步 |
|----------|------|--------|
| `PASS` | 所有检查通过 | 进入 Review 模型 |
| `FAIL` | 至少一项检查失败 | 编码模型修复或暂停 |
| `BLOCKED` | 环境/依赖问题 | 暂停，等待用户 |

**推荐值：**

| `recommendation` | 条件 |
|------------------|------|
| `READY_FOR_REVIEW` | 所有检查 PASS |
| `NEEDS_FIX` | 编译/测试 FAIL，可自动修复 |
| `NEEDS_HUMAN` | 边界违规或无法自动修复 |

---

## 5. Review 模型输出：review-guide.md

Review 模型**不重新验证**，只读取 `change-log.json` 和 `verification-report.json`，生成 Review Guide：

```markdown
# Review Guide: [任务名称]

> **Auto-generated by Review Model**
> **Task:** [任务名称]
> **Date:** 2026-06-23
> **HEAD:** [head_after]

---

## 1. 变更摘要（来自编码模型）

| 文件 | 类型 | 说明 |
|------|------|------|
| `app_state/mod.rs` | modify | 添加 `current_streaming_session` 字段 |

## 2. 验证结果（来自测试模型）

| 检查项 | 状态 | 详情 |
|--------|------|------|
| 编译 | ✅ PASS | 0 warnings |
| 测试 | ✅ PASS | 15/15 |
| 边界 | ✅ PASS | 无违规 |

## 3. 设计决策（来自编码模型 notes）

- 所有修改在 `can_modify` 范围内

## 4. Review 建议

- 无特别关注点，变更范围清晰
```

**Review 模型职责：**
- 读取结构化输出，格式化为人可读文档
- 标记编码模型的 `uncertainties` 为 Review 重点
- 不执行任何验证命令
- 不重新检查边界

## 6. 执行流程

```
┌─────────────────┐
│   编码模型       │
│  读取 Taskfile   │
│  执行编码        │
│  输出 change-log │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   测试模型       │
│  读取 change-log │
│  执行验证命令    │
│  输出 verification-report │
└────────┬────────┘
         │ PASS
         ▼
┌─────────────────┐
│   Review 模型    │
│  读取 change-log │
│  读取 verification-report │
│  输出 review-guide │
└────────┬────────┘
         │
         ▼
      [完成]
```

**暂停条件：**

| 阶段 | 暂停条件 | 处理 |
|------|----------|------|
| 编码 | 触及 `no_touch` 边界 | 立即停止，报告违规 |
| 编码 | `confidence: "low"` | 继续但标记为需要关注 |
| 测试 | 编译 FAIL | 尝试自动修复，失败则暂停 |
| 测试 | 测试 FAIL | 暂停，等待修复 |
| 测试 | 边界违规 | 暂停，必须人工确认 |
| Review | 无暂停 | 始终完成 |

## 7. 与现有体系集成

```
Heavy Model（架构决策）
    ↓
Taskfile + 上下文
    ↓
Lightweight Agent（编码 → 测试 → Review）
    ↓
Review Guide（人工快速确认）
    ↓
合并 / 继续
```

**适用场景：**
- 边界清晰的编码任务
- 已有明确修改范围
- 无需架构决策
- 可在单会话完成

**不适用场景：**
- 需要架构设计
- 跨模块协调
- 边界不清晰
- 需要用户交互决策
