## §1. 无代码解释版（给零基础读者）

> 用日常语言解释"做什么、为什么、影响什么"。

### 1.1 整个项目在做什么（一句话）

`northhing` 是一个 AI 编程助手桌面应用。每次你和 AI 对话时，AI 都要看一大段"系统提示"（告诉 AI 怎么工作）和"用户上下文"（你的工作区信息）。这俩内容**几轮对话内不会变**，所以应用把它们**缓存**起来重复用，省钱省时间。

### 1.2 最近刚做完的（避免重复造轮子）

- **`prompt-cache-stats`**（已完成）：给缓存加了一个"小账本"，记录每次查找是"命中"（直接用缓存）还是"未命中"（要重新算）。
- **`prompt-cache-stats-api`**（已完成）：给账本加了"算账方法"——能算"命中率"，能一键清零。

### 1.3 接下来 5 个 Task 是什么（用日常语言）

| # | 任务名 | 做什么 | 为什么 | 多久 |
|---|--------|--------|--------|------|
| **1** | 序列化账本 | 给账本加"导出 JSON"按钮 | 以后想看缓存效果时，能把数据发给监控系统 | 15 分 |
| **2** | 总命中率 | 给账本加"算总账"——不分子账本，只看一个大数字 | 一眼看出整个缓存系统好不好用 | 15 分 |
| **3** | 效果报告 | 把上面所有数字打包成一个"快照"，一次性拿走 | 调试/汇报时方便，不用一个个查 | 25 分 |
| **4** | 缓存层补考 | 给另一段代码（提示拼装器）加 3 个测试题 | 关键代码没测过，万一出 bug 难发现 | 30 分 |
| **5** | shell 命令 mock | 给"执行 shell 命令"功能加"假执行"模式 | 以后测试时不需要真跑命令，更快更安全 | 45 分（**自动体检门控**） |

### 1.4 影响范围（**重要承诺**）

- **任务 1-3**：只动 `prompt_cache.rs` 一个文件，**纯加法**（只新增方法，不改任何已有方法）
- **任务 4**：只动 `partitioned_loader.rs` 的 `#[cfg(test)]` 测试模块，**生产二进制零变化**
- **任务 5**：只动 `command.rs` 的 `#[cfg(test)]` 测试支持模块，**生产二进制零变化**

**承诺：跑完这 5 个 Task，整个应用的所有现有测试 100% 仍然通过。**

## §2. 人类决策清单（reviewer 必读，1 分钟审批）

### 2.1 整体审批项（跑 5 个 Task 之前一次性决定）

| 决策点 | 选项 | 默认（推荐） | 影响 |
|--------|------|-------------|------|
| 任务执行顺序 | A. 1→2→3→4→5（顺序）<br>B. 只跑 1-3（保守）<br>C. 跳过 5（看下游） | **A** | A 跑完所有 5 个；B 跳过 4-5；C 跳过 5 |
| Task 5 是否值得做 | 值得 / 跳过 | **自动决定**（见 `execution.md §4.5` 步骤 0） | 体检通过则跑，否则 SKIP |
| review 频率 | 每个 Task 后 / 全部跑完后 | **每个 Task 后** | LAEP 协议本来就要求每 Task 一份 review-guide.md |

### 2.2 紧急刹车（什么情况下人类喊停）

- 任何 Task 改坏了现有测试 → **LAEP Testing Model 报 FAIL** → 自动回到 Coding 修复，**3 次失败后 STOP 报告人类**
- Task 4 的体检发现 `partitioned_loader.rs` 没有 dev-dep `tokio` → **自动降级**到 `prompt_builder_impl.rs` 写同步测试
- Task 5 的体检发现 `tool-execution` 已有自己的 mock → **自动 SKIP** Task 5

### 2.3 不需要人类批准的"决策"（LAEP 4B 可自主判断）

- 选哪个 cargo 命令来跑测试（Plan 已固化）
- 选哪个测试函数名（Plan 已给具体名字）
- 选哪种 JSON 结构（Plan 已给具体字段）
- 选哪种错误处理方式（沿用现有 `Arc<Mutex<T>>` + `unwrap()` 模式）
