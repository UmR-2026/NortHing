# northhing 可配置开发体验产品需— > 范围：定— northhing 加载任意外部目标项目后的产品定位、目标用户、核心体验、产品规格、关键边界、平台入口差异、功能需求和成功指标— > 关联文档：[external-research.md](research/external-research.md) 提供外部证据，[design.md](design.md) 承接架构设计，[implementation-plan.md](implementation-plan.md) 承接阶段落地，[traceability-matrix.md](traceability-matrix.md) 承接覆盖关系检查— ## 1. 背景

AI 编码工具正在从代码补全、单轮问答和单点审查，演进为覆盖理解、修改、验证、审查、发布和复盘的智能体开发体验。northhing 需要同时服务两类场景：

- 质量保障要求较低的任务、探索性改动、演示原型和文档改动：默认简洁快速— 团队协作、核心路径、权限、网络、发布、迁移、合规和受管项目：按配置、风险和阶段显露证据、审查、门禁、风险接受和审计— ## 2. 产品目标

| 目标 | 产品要求 |
|---|---|
| 默认快— | 无显式规则时，用户能直接开始任务并获得简洁结果摘— |
| 风险可解— | 风险、检查、审查和阻断都说明来源、后果和下一— |
| 治理可配— | 高强度审查、强制检查、门禁和审计由项目、路径、团队、组织或用户选择启用 |
| 安全常驻 | prompt 注入、hook、MCP、网络、凭据、删除、跨目录写和发布凭据始终走安全边— |
| 沙箱边界可见 | 用户能分清本机、远程、容器、ACP、MCP、插件和浏览— 桌面动作的执行位置、隔离等级和授权范围 |
| 阶段收益明确 | 每个落地阶段都先交付短而明确的用户收益，技术前置和延期边界必须可解— |
| 复杂能力后台— | 证据包、交付物图谱、质量数据面和评测只— PR、团队、发布、事故或合规场景显性化 |
| 生态扩展预— | 先稳— Kernel 扩展点；OpenCode 等插件生态后续接入时必须— northhing 边界内生— |
| 体验可度— | 速度、打断、误升级、安全确认、用户覆盖和质量结果按用户画像切— |

## 3. 用户画像

| 用户画像 | 默认体验 | 典型升级触发 | 关键保护 |
|---|---|---|---|
| 快速试验— | 无配置启动，直接完成低风险任务，结束时展示简洁摘— | 高权— shell、网络、凭据、删除、跨目录写或准备分享 | 安全边界常驻，质量建议可跳过 |
| 独立开发— | 自动复用 README、脚本和仓库规则，给弱提示和可选检— | 准备 PR、跨模块改动、验证缺— | 未验证项和信心摘要清晰可— |
| 团队开发— / 审查— | 准备 PR 或进入团队规则时展示就绪度和规则来源 | 受保护分支、CODEOWNERS、路径规则、团队配— | 审查强度可配置，阻断来源可追— |
| 技术负责人 / 架构— | 关键变更展示影响候选、验证缺口和风险接受入口 | API、数据、权限或架构边界变化 | 影响候选、证据引用和风险接受分离 |
| 安全 / 合规负责— | 受管场景展示批准、审计、发布和回滚证据 | 凭据、网络、MCP、hook、生产发布、受管项— | 组织策略高于本地覆盖 |
| 平台 / 插件维护— | 主动配置以待确认能力展示，确认后再参与执— | hook、plugin、MCP、自定义工具、工具复写或远程执行面变— | 主动配置先作为未授权候选，隔离后授— |

用户画像用于选择默认入口、解释语言和提示密度。内部策略画像不是用户侧主概念。体验呈现应聚焦任务状态、弱提示、确认、受限原因、设置项和下一步— ## 4. 核心体验路径

| 路径 | 触发 | 体验呈现 | 系统动作 |
|---|---|---|---|
| 快速开— | 质量保障要求较低、低风险任务、个人项目、探索性改— | 任务结果、已运行检查、未验证项、下一步建— | 轻量项目理解，保持低干预策略 |
| 上下文保— | 核心路径、权限、网络、迁移、发布、验证缺— | 风险原因、推荐检查、跳过后果、信心摘— | 使用弱提示、可折叠摘要或定向确— |
| 团队治理 | 团队配置、受保护分支、路径规则、PR 协作 | PR 就绪度、审查建议、风险接受入— | 按团队配置投影证据引用和检查要— |
| 合规 / 发布 | 合规、生产发布、受管项目、事故复— | 批准、审计、发— 回滚证据、完整证据包 | 按合规策略保留完整追— |
| 执行安全 | 高权— shell、网络、凭据、MCP、hook、删除、跨目录— | 安全确认、范围、后果、可选隔离路— | 安全边界独立判定 allow/ask/deny/应急放— |

## 5. 产品规格

| 规格 | 用户可见结果 | 产品约束 |
|---|---|---|
| 项目打开 | 项目类型、主要入口、规则来源、可运行命令和风险提— | 无配置时只做轻量推断；推断结果默认用于提示，不直接形成强制策— |
| 任务执行 | 当前任务状态、关键动作、必要确认和可选检— | 低风险任务不要求用户先理解治理概— |
| 任务完成 | 改动摘要、已验证项、未验证项、残余风险和下一— | 默认摘要短；证据引用只在 PR、团队、发布、事故或合规场景显露 |
| 内部策略画像 | 策略原因、作用范围、展示层级和可覆盖选项 | 内部画像按任务和动作计算，不作为用户侧标签，不把整个项目永久贴上单一模式 |
| 提示与确— | 静默执行、行内弱提示、可折叠摘要、特定场景确认、阻断原— | 默认不用弹窗；同类低价值提示应合并、延后或进入任务结束摘要 |
| 检查建— | 推荐检查、不可运行原因、替代验证和跳过后果 | 推荐检查不等同阻断；强制检查必须有配置、确定性失败或用户显式升级来源 |
| 安全确认 | 命令、文件、目录、域名、凭据、权限范围和后果 | 安全确认独立于质量治理，且优先用沙箱、白名单和范围限制降— |
| 沙箱与执行位— | 本机/远程/容器/ACP/MCP/插件/浏览器动作的位置、隔离等级组合、降级原因和可选替代路— | 只有真实隔离存在时才展示“沙箱允许”；权限确认不能包装成系统级沙箱 |
| 阶段性收— | 当前阶段用户能得到的体验变化、必要技术前置、延期边界和质量一致性说— | 不用完整模块边界代替阶段收益；后台技术能力必须说明服务哪个用户场— |
| 用户覆盖 | 单次、会话、任务或工作树（worktree）范围，带期限和撤销入口 | 覆盖只能降低建议噪音或接受质量风险，不能绕过安全拒绝和组织拒绝策— |
| 团队配置 | 配置来源、优先级、— 突状态和生效范围 | 团队配置需要可解释；— 突规则先提示或降级，不静默选择高风险路— |
| PR 就绪— | 可复制到 PR 的变更、验证、风险和证据引用摘要 | 只有准备 PR 或团队策略启用时显露，不干扰普通任务中间过— |
| 复杂能力入口 | 证据包、交付物图谱、发布就绪度、事故回溯和评测入口 | 复杂能力从用户任务触发，不作为默认首屏或默认流程 |
| Kernel 扩展— | 生命周期 hook、事件、打点、工具复写候选和插件效果候— | 先提供空实现和稳定契约；外部插件运行时不作为默认能力 |
| 工具复写契约 | 被复写工具、复写来源、权限范围、持续时间和撤销入口 | 用户确认只表达意图；实际文件、shell、网络和凭据访问仍由安全边界执行 |

### 5.1 实际产品变更清单

| 产品入口与组— | 变更 | 默认状— | 触发条件 |
|---|---|---|---|
| 项目上下文条 | 展示项目名、执行位置、主要语言/包管理器、规则来源和远程状— | 常驻但紧— | 打开项目、切— worktree、进入远程工作区 |
| 任务结果摘要 | 在任务结束处展示改动、已运行检查、未验证项、残余风险和下一— | 默认展示短摘— | 每次任务完成或用户准备复制结— |
| 行内风险提示 | 在相关消息或工具调用旁展示风险原因、推荐检查和跳过后果 | 弱提示，不弹— | 中风险路径、验证缺口、跨模块改动 |
| 安全确认面板 | 展示命令/文件/域名/凭据、授权范围、执行位置、沙箱等级、后果、允许一— 会话/拒绝/隔离选项 | 仅安全动作触— | shell、网络、凭据、删除、跨目录写、发— |
| 沙箱状态块 | 展示本次动作是否为权限确认、快— worktree 隔离、只读范围、网络限制、进程隔离或容器隔离 | 默认并入安全确认或任务头 | 远程、ACP、MCP、插件、未知网络、批量写— |
| PR 就绪度面— | 展示变更摘要、验证状态、未验证项、风险接受、证据引用和复制— PR 按钮 | 默认隐藏 | 用户准备 PR、检测到受保护分支或团队策略 |
| 团队规则说明 | 展示规则来源、优先级、— 突和生效路径 | 折叠展示 | 规则— 突、受管路径、强制检— |
| 主动配置信任详情 | 展示 hook/plugin/MCP/custom 工具来源、hash、权限、触发条件和信任状— | 发现后提示，默认不执— | 新增或变更主动配— |
| 远程上下文提— | 展示执行主机、工作区根、端— 网络范围、路径映射和不可用能— | 远程项目常驻紧凑提示 | SSH、容器、Dev Container、远程工作区 |
| 设置页扩— | 提供提示密度、默认授权范围、团队规则来源、应急放行记录和插件契约开— | 不打断主流程 | 用户主动进入设置或审计场— |

### 5.2 页面与面板布局

| 入口 | 布局要求 |
|---|---|
| 桌面工作— | 顶部项目上下文条；中间保留聊— 任务主流；右侧使用可折叠上下文与就绪度面板；底部继续承载终端/工具输出。普通任务不进入独立治理仪表— |
| 命令— / 终端 | 使用短状态块：项目、执行位置、安全范围、已验证/未验证、下一步。确认问题用单行选项和编号，避免长表格阻— |
| PR / 代码托管平台 | 生成可复— Markdown 块：变更、验证、未验证、风险接受、证据引用。默认不创建强制检— |
| 远程 / 云端 | 在任务头和安全确认中固定显示执行位置、路径映射和网络边界。无法执行的本地能力显示替代路径 |
| 设置 / 审计 | 把团队规则、用户覆盖、应急放行、主动配置信任、扩展点事件作为可查看记录，不进入默认首— |

### 5.3 关键用例与体验增— 这里的质量要求指当前任务需要的验证、审查和证据保障强度，不表示允许低质量交付— | 用例 | 质量要求 | 触发条件 | 体验变化 | 用户收益 | 验收— |
|---|---|---|---|---|---|
| 质量保障要求较低的探索性改— | — | 用户明确追求快速试验，且未触发安全高风险或团队强策— | 保留紧凑项目上下文条和任务结束短摘要；不展示 PR 就绪度、完整证据包或审查建— | 更快获得首个可用结果，减少与任务目标无关的治理负— | 首次有用动作耗时可采样；无多余弹窗；摘要包含已验— 未验证项 |
| 文档或小范围非关键改— | 低到— | 变更集中在文档、文案或非核心路径，缺少验证但风险有— | 行内显示已改文件、未运行检查和可选验证；推荐检查不默认阻断 | 用户能保持写作或小改动流畅，同时知道未验证风— | 提示可忽略、可展开、可复制；跳过检查会进入任务摘要 |
| 未知网络命令或外部访— | 安全高敏 | shell、网络、凭据、删除、跨目录写或发布动作触发安全边界 | 弹出安全确认面板，展示命令、域名、目的、授权范围、后果和隔离选项 | 用户能快速判断是否允许执行，并控制授权范— | 用户确认后只在声明范围内生效；拒绝或隔离路径可继续任— |
| 准备 PR 或分享变— | 中到— | 用户准备提交 PR、复制结果，或检测到受保护分— 团队策略 | 按需显示 PR 就绪度面板，列出变更、验证、未验证、风险和证据引用 | 减少审查人追问，用户不用手动整理验证和残余风— | 就绪度不在普通任务中间过程显露；PR 摘要可复— |
| 受管路径或团队规则命— | 团队定义 | 变更命中 CODEOWNERS、CI、路径规则、受保护分支或组织策— | 折叠展示团队规则来源、优先级、强制项和— 突处— | 用户知道为什么需要额外检查，并能定位负责人或规则来源 | 阻断来源可追踪；风险接受和强制检查有记录 |
| 远程或云端执— | 按任务风— | SSH、容器、Dev Container、远程工作区或云端异步任— | 在任务头和安全确认中固定显示执行主机、路径映射、端— 网络边界和不可用能力 | 用户能分清本地与远程能力差异，减少误操作和无效验— | 不可支持能力有替代路径；网络和端口范围可确认 |
| 新增主动配置 | 安全高敏 | hook、plugin、MCP、自定义工具或工具复写被发现— hash 变化 | 显示主动配置信任详情，提供禁用、仅记录、信任当— hash、查看差— | 用户能先理解来源和权限，再决定是否让配置生效 | 默认不执行；hash 变化后重新确认；信任状态可撤销 |

## 6. 关键边界

| 边界 | 产品判定 |
|---|---|
| 内部策略与用户语言 | 内部策略画像可以存在；用户侧不直接学习内部模式名，只看到自然语言状态、原因和操作 |
| 质量治理与执行安— | 质量治理回答“是否足够可靠”，执行安全回答“是否允许执行”。两者分开展示、分开授权、分开记录 |
| 建议与阻— | 默认提供建议、摘要和跳过后果；阻断只来自安全拒绝、组织策略、确定性失败或明确强制策略 |
| 弱提示与弹窗 | 默认使用行内提示、可折叠摘要和任务结束汇总；弹窗仅用于安全确认、不可逆操作、组织策略或用户明确进入关键流程 |
| 用户覆盖与组织策— | 用户可以临时选择更快或更严；受管组织策略、受保护分支和安全拒绝保持更高优先级 |
| 项目规则与主动配— | AGENTS、CONTRIBUTING、CODEOWNERS、CI 等规则可作为项目知识；hook、plugin、MCP 和自定义工具先进入信任审— |
| 插件兼容与核心模— | OpenCode 等生态后续通过适配器接入；northhing 内部只依赖规范事件、权限和插件效果候选契— |
| 工具复写与用户确— | 用户可启用工具复写，但复写必须绑定项目、来源、hash、权限范围和期限，不能静默改变全局工具语义 |
| 模型判断与系统状— | 模型可生成候选风险、摘要和建议；策略状态、验证结果、授权状态必须来自确定性事件或用户决策 |
| 本地能力与远程能— | 本地可直接感知文件、命令和环境；远— 云端必须显式标注工作区、权限、环境缺口和不可支持动作 |
| 质量保障要求较低的任务与受管任务 | 质量保障要求较低的任务默认低摩擦；受管任务按配置显露审查、审计、批准和门禁 |

## 7. 平台与入口差— | 入口 | 默认体验 | 差异要求 |
|---|---|---|
| 桌面本地 | 文件、终端、Git 和项目规则整合在同一工作— | 高权限命令、跨目录写、网络和凭据访问需要清晰确认；本地不可运行检查要给替代验— |
| 命令— / 终端 | 以文本摘要、确认提示和命令结果为主 | 输出必须短、可复制、可继续；没有富界面时仍要保留任务状态、未验证项和安全范围 |
| 远程开— | 通过 SSH、容器、Dev Container 或远程工作区执行文件、命令和 Git 操作 | 必须标注执行主机、工作区根、权限范围、网络边界、端口暴露和本地/远程路径差异 |
| Web / 远程控制— | 通过远程适配器管理任务、diff、验证和审批 | 不假设本地路径或本机依赖；不支持的本地能力必须显示原因和可替代路— |
| 云端异步任务 | 以任务计划、diff、验证结果和批准为中— | 长任务减少中途打断；需要在执行前或关键阶段收集授权，完成后给可审查摘要 |
| PR / 代码托管平台 | 以就绪度摘要、检查状态、评论和风险接受呈现 | 默认不创建强制检查；只有团队或组织策略启用时才投影为门禁 |
| 受管企业环境 | 以组织策略、审计、批准和合规证据为中— | 组织拒绝策略不可被本地覆盖；所有放行都需要范围、操作者、原因和期限 |
| 插件 / MCP / hook 入口 | 作为能力扩展和主动执行面 | 默认最小权限；先展示来源、hash、权限、触发条件、工具复写和信任状态，再允许在对应执行域内生效 |

## 9. 配置优先— ```text
组织拒绝 / 受管强制要求
 > 安全边界
 > 最近确认的路径或团队规— > 工作区配— > 允许范围内的会话/任务覆盖
 > 全局/用户默认— ```

用户覆盖可以降低建议噪音、跳过推荐检查或请求风险接受。安全边界、组织拒绝策略、受管路径强制策略和受保护分支规则保持更高优先级— ## 10. 成功指标

## 11. 版本边界

| 阶段 | 用户可见收益 | 后台前置和延期边— |
|---|---|---|
| P0 | 低风险任务能快速完成，并看懂项目入口、安全范围、已验证/未验证项和下一— | 只做轻量项目理解、基础安全决策和最小事件；完整证据包、图谱和团队门禁延期 |
| P1 | 风险、远程能力缺口、沙箱降级和推荐检查变得可理解 | 需要风险分类、提示降噪、sandbox capability 事件和扩展点空实现；不执行外部插件运行时 |
| P2 | 准备 PR 或命中团队规则时获得可复制摘要、规则来源和风险接受入口 | 需要规则来源合成、PR 投影和证据引用；强制检查只在配置或确定性风险下出现 |
| P3 | 受管项目的需求、开发、验证、审查、发布和风险接受强度保持一— | 需要路径级强策略、证据包投影、最小图谱和发布/事故事件；普通任务仍保留轻量路径 |
| P4 | 用真实任务回放和指标持续优化治理策略，并淘汰无收益能— | 需要评测血缘、策略版本、成本采样和保留集；生态适配以不改变内核权威状态为前提 |
